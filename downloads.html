NEW!!! - MEET - a java app that brings together mongoDB and Twitter to insert all your recieved tweets up into mongoDB to allow for further analysis - map reduce queries etc...., a simple way to start to take your company twitter account - tweets and start building some real life analytical value for your company.

1.) Extract all files to root folder using winrar or similar

2.) The folders for the mongo library <.com, meta-inf, .org> are all included

3.) Download twitter4j library folder from the website twitter4j-4.0.0

4.) Add twitter4j-4.0.0 to your classpath i.e: add to environment variables if in windows and .profile if unix if using windows as .;C:\TwitterJava\twitter4j-4.0.0\lib\twitter4j-core-4.0.0.jar

5.) Modify jsiconfig.txt with your mongodb details

6.) Modify twitter4j.properties file with your consumer and access token keys - you will need to register your app on twitter and follow instructions if you have not done this previously

7.) Compile code javac meet.java and run as appropriate

8.) Note this will take the latest 20 tweets and post them up into mongo

9.) You can easily adjust this code to filter and loop throughout the delay as a daemon process etc.

10.) Please Note execute the code using java meet -i-d note: -i-d flags remove -d = disables display, -i = disables inserts to allow testing and so just run java meet will display and insert the latest 20 tweets up to your mongoDB instance

11.) Enjoy....

JSIMONGO - java software that will allow realtime data entry into a mongoDB, you will need to download and adjust the sample JSON and adjust the parameters to produce a configured random set of time lapsed data.

This is very much a pre-beta piece of code at the moment but I wanted to get this out here as I believe it may well be very useful.

It has been built using java version "1.7.0_51" but will work without issue on previous versions back to JDK v1.6.

It has been built using the official mongo drivers mongo-java-driver-2.11.4-sources.jar, included in the jar file.

To modify follow the below instructions to help you get started.

1.) Download the jsi.jar - this holds the main class files

2.) Download the sampleConfigFiles.rar and extract - this provides the main jsiconfig.txt and seed data that needs modified

3.) Edit the jsiconfig.txt to point at your mongoDB Lab database you can set this up easily by going to Mongolab.com

4.) Edit the seed files the prodlist and pricelist need to mirror each other in number of lines

5.) Edit the hourweight list - this simulation runs on an hour by hour varaince and this gives a product hour weighting ie: the more products sold in 1 hour the higher the weighting.

6.) Edit dateslist for number of days to run this for, just one date will do at first

7.) Edit entitylist for the number of entities to run this for again this can be left at one initially but this can simulate for example a number of company sites / shops etc.

8.) Finally you can test by running the code using java -jar jsi.jar -i

9.) Note: There are two options here -i means do not insert into mongodb selected and -d means do not display transactions to the screen. Just the -i option is shown above and these are completely interchangeable.

How to notes

Amazon Web Service cloud Server setup for a Web Domain

1.) If you do not yet own a web server domain eg:myowndomain.co.uk - go to https://uk.godaddy.com/ or similar provider and purchase a new domain the cost varies but aprox 2.99 cost for 1-2 years

2.) Now you have a domain go to https://aws.amazon.com/ you will have to signup but go for the AWS Free tier if this just personal or for trial use. Select the Amazon EC2 Service but have a look around and see what fits your needs.

3.) Select and create your instance - I selected an Ubuntu instance.

4.) Launch instance using the AWS console. Note you will now have an instance id.

5.) I suggest you setup an Elastic IP as otherwise if you need to restart you instance your IP will change each time. My instance has been running now for ages without any issue but very useful at the start. This essential tags an IP against the instance id provided

6.) logon using putty - each different instance has a different standard username to logon through your account eg: my ubuntu instance was as simple as ubuntu and use port 22 as SSH

7.) You will need to change security settings in order to transfer files onto the account using products like the FileZilla client for example.

8.) Once you are happy that you can logon an off it it a simple matter of installing a webserver rg: Apache2 and starting this service. Loads of examples how to do this on google. Simple as logon and execute 'sudo apt-get install apache'.

9.) Check apache is running by executing ps -ef | grep apache - you should get 4 or 5 lines returned

10.) Run the next commands as sudo go to the /var/www/ folder and create a folder named to your website eg: northmount.org and so you should have a folder /var/www/northmount.org

11.) Copy your index.html into this folder as the startup page for your website

12.) restart apache2 by executing 'sudo service apache restart, now go back to your godaddy.com domain provider and create a zone file that points to your Elastic IP - IP Address. You can use www.whatsmydns.net to check if this change as been propogated this can take 45mins or so. Then logon to your domain from the internet.

Enjoy full control and a very cost effective web server with added benefit of your own 24*7 Server.

How to Notes for GITHUB

1.) Create your github account using the github website and noting credentials

2.) login to your favorite debian based distro - either debian or ubuntu and mkdir the folder than you are going to develop in

3.) Install git for linux by running sudo apt-get install git

4.) Go to your git hub web account in your browser and create a repository to hold your code, note down the url for this repository https://github.com/yourreponame..

5.) Go to your linux folder and type 'git clone https://github.com/yourreponame....'

6.) The above will of created a folder repository where you can start creating your code, and once your ready you can run these three commands to push these changes back into your repository. a.) git add . b.) git commit -m "your commit message" c.) git push

7.) Check the repository through your browser and you will now see your changes recorded in your new github repository

8.) In addition if you ever want to complete a pull of code again from the repo you can run the following code a.)git checkout b.) git pull

How to Notes for Setting up an AWS Kinesis Stream

This comes in two parts, setup and then configure. Kinesis basic architecture is as producer that adds data onto a stream and application or consumer that then displays this or moves this into a nosql db or whatever for analysis.

SETUP

1.) Create AWS account

2.) In AWS Create an internal AWS User with full rights

3.) Download the aws_access_key_id, aws_secret_access_key

4.) Use the template credentials file provided in the aws-java-sdk-1.8.5 in folder samples\AmazonKinesis and paste the access keys into this file.

5.) Save the credentials file in folder c:\users\insert your user\.aws\

6.) Download from GIT the jsikinesis library and unzip into appropriate folder

7.) You will need have JDK 1.6 or higher - when compiling it does give a warning that code is for 1.6 but ignore check that java -version works in the folder

8.) Make sure your class path points to the relevant folder, example given below for windows env vars, I basically made sure the code pointed at all the aws-java-sdk jar files.

My CLASSPATH example = .;C:\jkinesis\com\aws-java-sdk-1.8.5\lib\aws-java-sdk-1.8.5.jar;C:\jkinesis\com\aws-java-sdk-1.8.5\lib\aws-java-sdk-1.8.5-javadoc.jar;C:\jkinesis\com\aws-java-sdk-1.8.5\lib\aws-java-sdk-1.8.5-sources.jar;C:\jkinesis\com\aws-java-sdk-1.8.5\lib\aws-java-sdk-flow-build-tools-1.8.5.jar;C:\jkinesis\com\aws-java-sdk-1.8.5\third-party\aspectj-1.6\aspectjrt.jar;C:\jkinesis\com\aws-java-sdk-1.8.5\third-party\aspectj-1.6\aspectjweaver.jar;C:\jkinesis\com\aws-java-sdk-1.8.5\third-party\commons-codec-1.3\commons-codec-1.3.jar;C:\jkinesis\com\aws-java-sdk-1.8.5\third-party\commons-logging-1.1.1\commons-logging-1.1.1.jar;C:\jkinesis\com\aws-java-sdk-1.8.5\third-party\freemarker-2.3.18\freemarker-2.3.18.jar;C:\jkinesis\com\aws-java-sdk-1.8.5\third-party\httpcomponents-client-4.2.3\httpclient-4.2.3.jar;C:\jkinesis\com\aws-java-sdk-1.8.5\third-party\httpcomponents-client-4.2.3\httpcore-4.2.jar;C:\jkinesis\com\aws-java-sdk-1.8.5\third-party\jackson-annotations-2.1\jackson-annotations-2.1.1.jar;C:\jkinesis\com\aws-java-sdk-1.8.5\third-party\jackson-core-2.1\jackson-core-2.1.1.jar;C:\jkinesis\com\aws-java-sdk-1.8.5\third-party\jackson-databind-2.1\jackson-databind-2.1.1.jar;C:\jkinesis\com\aws-java-sdk-1.8.5\third-party\java-mail-1.4.3\mail-1.4.3.jar;C:\jkinesis\com\aws-java-sdk-1.8.5\third-party\joda-time-2.2\joda-time-2.2.jar;C:\jkinesis\com\aws-java-sdk-1.8.5\third-party\spring-3.0\spring-beans-3.0.7.jar;C:\jkinesis\com\aws-java-sdk-1.8.5\third-party\spring-3.0\spring-context-3.0.7.jar;C:\jkinesis\com\aws-java-sdk-1.8.5\third-party\spring-3.0\spring-core-3.0.7.jar;C:\jkinesis\com\aws-java-sdk-1.8.5\third-party\stax-api-1.0.1\stax-api-1.0.1.jar;C:\jkinesis\com\aws-java-sdk-1.8.5\third-party\stax-ri-1.2.0\stax-1.2.0.jar

9.) That's you done! for setup.

CONFIGURE

1.) Download from products page jsiKinesis, to compile to your folder type "javac jsiKinesis.java" and then to run "java jsikinesis"

2.) Note: just to test a run you can add -i or -d flags to the end -i = excludes inserts, -d = excludes display"java jsikinesis -i" for example will display but will not insert, very handy for kinesis.

3.) There are a couple of other utilities you will need just to list and remove streams a.) java AmazonKinesisList = lists all your streams you have available b.) java AmazonKinesisDelete = deletes all your streams - amazon charges a small amount for open streams so worth doing

4.) FINALLY java AmazonKinesisGet retrieves the data from your stream [configure your stream name in here and match with the one in jsikinesis.java] unless just keep it as philsteststream in the meantime! Note this runs for ever while there are streams so either CTRL - C out or run the java AmazonKinesisDelete code and then you will see it stop.

NOTE: This is just configured to 1 shard per stream at the moment, but not that tricky to expand. As you optimize a real scenario more complexity starts as you get to split and merge shards on the fly depending on throughput required.

Just to demo - its nice to bring up 2* terminals/consoles 1 with the producer running and 1 with the consumer app. Also go to the AWS Console and it gives you a nice breakdown of the streaming analysis. Start the producer 1st as that creates the stream and shard and then once streaming started, bring up the consumer.

Thats you done, you have a Kinesis stream going, you can easily point the data towards your RDBMS/NOSQL database etc for further analysis
